# Agift — 数据准备说明

## 文件
- `brands.csv` — 80 个常用送礼品牌（大牌 / 文艺小众 / 当下流行 / 生活方式 / 美妆 / 数码 / 文具）
- `categories.csv` — 25 个常见送礼品类，含场合、风险等级、备注
- `items.csv` — 136 个具体单品，按品类 + 品牌交叉，覆盖四档价格

三张表通过 `brand_id` / `category_id` 关联，方便后续直接导入到 App 数据库。

## 价格分档（我的解读）
你给了 4 档，但有空隙，我按以下方式归并：

| Tier | 区间 | 适用场景 |
|------|------|---------|
| T1 | ¥100–200 | 小心意、伴手礼、同事 |
| T2 | ¥200–1000 | 朋友、生日常规礼 |
| T3 | ¥1000–3000 | 轻奢、好友/家人重要场合 |
| T4 | ¥3000+（重点 ¥5000+）| 大礼、长辈、爱人、商务重客 |

如果你想让分档严格按你说的 100-200 / <1000 / ~2000 / >5000，我可以重新切，告诉我即可。

## 几个判断
1. **价格是参考值**，写的是国内官方/常见渠道大致价位，随汇率和促销浮动，App 上线前建议接价格 API 或人工 review。
2. **男女标签**用 `recipient` 列写了，但很多品类是中性的（香水、蜡烛、茶、酒、数码、文具）。App 里建议做成多选 tag。
3. **风险等级** `risk_level` 是我加的一个维度——某些品类（戒指、包、香水）需要对对方喜好有相当了解才不会送错；某些（蜡烛、Aesop、巧克力）几乎不会出错，可以作为"不熟的人也能送"的安全选项。
4. **文化场景**没拉单独表。如果 App 想做"中式 vs 西式"的差异（比如送领导/长辈的中式礼 vs 送女朋友的西式礼），需要再加一张 `occasions.csv` 或在 categories 上扩展。

## 下一步可以讨论的方向
- **决策树是什么形状**：用户先选"送给谁"还是先选"预算"？我猜大多数人是从"我要花多少钱"或"为什么场合"切入。
- **过滤维度**至少四个：predicted_recipient（性别/年龄/关系）× occasion × budget × style（极简/张扬/老钱/潮流）。要不要也建一张 `recipient_personas.csv`，比如"30岁知性女友/50岁岳母/同事女生/男领导"这种典型人设？
- **数据来源**：目前是我凭常识写的。如果想让数据更可靠且持续更新，可以考虑：
  - 接小红书/得物的热门礼物榜
  - 抓品牌官网的当季新品
  - 人工维护一份"季度推荐位"
- **App 形态**：是问答式（像测一测）、列表浏览式（像逛淘宝）、还是 AI 对话式（描述对象→推荐）？这会决定数据怎么再加工。

让我知道你想往哪个方向走，我们再细化下一层。
